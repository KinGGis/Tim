# Playbooks de workflow

Chaque rôle dispose d'un fichier de checklist + templates :

1. `playbooks/po.md` - briefing client, critères d'acceptance, template d'issue backlog.
2. `playbooks/ba.md` - structure des specs fonctionnelles/techniques, matrice de tests.
3. `playbooks/dev.md` - prompts Copilot, séquences de génération/refactor/tests.

Les playbooks s'appuient sur des dossiers `prompts/` et `automation/` pour relier outils et CI.
